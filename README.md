# ETL Mini Pipeline

A reproducible pandas-to-SQLite data quality pipeline with synthetic operational records.

## Portfolio purpose

Operational extracts often contain duplicate identifiers, inconsistent categories, malformed dates, invalid emails, missing fields, and numeric values stored as text.

## Approach

A pandas pipeline profiles raw rows, standardizes fields, validates business rules, separates rejected records, writes a clean CSV, and loads curated tables plus a quality summary into SQLite.

## Key artifacts

- `raw_data.csv` — intentionally messy synthetic source records
- `etl_pipeline.py` — extract, transform, validate, and load workflow
- `clean_data.csv` — validated output generated by the pipeline
- `DATA_QUALITY_RULES.md` — explicit acceptance and rejection rules

## How to run or review

```text
python etl_pipeline.py
```

Run commands from this project directory. All paths inside the code are relative.

## Design decisions

- Preserve raw input and create separate clean and rejected outputs.
- Make rejection reasons explicit instead of silently dropping invalid records.
- Use an idempotent SQLite replace load for a deterministic portfolio demo.

## Limitations

The pipeline is sized for a local demonstration. A production implementation would add orchestration, incremental loads, schema contracts, lineage, alerting, and stronger observability.

## Next iteration

Add unit tests, incremental ingestion, configurable quality thresholds, and a lightweight orchestration workflow.

## Data statement

Every record, identifier, organization, person, scenario, and result in this project is synthetic.
No employer, client, university, colleague, customer, credential, private path, or sensitive record is used.
